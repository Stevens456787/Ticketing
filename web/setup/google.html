Configure Sign in with Google

Use your agency Google Workspace tenant to enable OAuth2 sign-in for the Traffic Tickets app. Follow these setup tasks in the Google Cloud console.

  1. Create an OAuth client ID for a web application in the Google Cloud Credentials dashboard.
  2. Add the Traffic Tickets redirect URI (e.g., https://app.your-agency.gov/auth/google/callback) to the authorized redirect URIs list.
  3. Enable the "Google People API" so the app can fetch basic profile data after sign-in.
  4. Download the client ID and secret, store them in your secure secrets manager, and configure the backend auth service to use them.
  5. Return to the login page and toggle the "Sign in with Google" button to active once your backend deployment is complete.
Tip: If you enforce device quick-login, remind officers to complete one Google SSO flow on each new device before enabling the biometric shortcut.
